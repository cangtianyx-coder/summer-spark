# UX/UI 自查清单
# UX/UI Checklist

**版本:** MVP Build  
**最后更新:** 2026-04-22  
**适用平台:** iOS / Android / Web

---

## 一、布局与结构 (Layout & Structure)

- [ ] 页面无横向溢出（移动端 320px 起）
- [ ] 响应式断点正确（320px / 768px / 1024px / 1440px）
- [ ] 主要内容在首屏可见，无需滚动即可看到核心功能
- [ ] 布局采用统一栅格系统（12列或8列）
- [ ] 元素间距遵循 4px base grid（4/8/16/24/32/48px）
- [ ] 页面结构符合 F 型或 Z 型阅读习惯
- [ ] 移动端底部导航栏添加 safe area 适配（≥34px）
- [ ] 模态框/弹窗未覆盖系统导航区域
- [ ] 深色模式/浅色模式布局保持一致
- [ ] 页面存在空状态时，显示友好提示和引导

---

## 二、视觉与色彩 (Visual & Color)

- [ ] 品牌色、辅色、中性色、功能色已定义并文档化
- [ ] 主文本与背景对比度 ≥ 4.5:1（WCAG AA）
- [ ] 大文本（≥18sp）与背景对比度 ≥ 3:1
- [ ] 功能色语义一致（红色=错误，绿色=成功，黄色=警告）
- [ ] 色彩不作为传递信息的唯一方式（兼顾色盲用户）
- [ ] 无hover状态颜色与正常状态过于接近的问题
- [ ] 深色模式下对比度问题已测试
- [ ] 图标色彩与文字色彩协调统一
- [ ] 按钮、卡片等元素视觉权重与重要性匹配
- [ ] 选中/激活状态有明确的视觉区分

---

## 三、字体与排版 (Typography)

- [ ] 正文字号 ≥ 16sp（移动端）
- [ ] 标题层级清晰（h1 > h2 > h3 视觉差异明显）
- [ ] 行高设置合理（正文 ≥ 1.5，标题 1.2-1.3）
- [ ] 字体栈包含系统回退字体（-apple-system, Roboto等）
- [ ] 超长文本有截断处理或换行规则
- [ ] 无文字叠加在图片或复杂背景上（可读性差）
- [ ] 英文内容正确使用连字符和避头尾规则
- [ ] 数字、字母、汉字混排时排版协调
- [ ] 引用块/代码块使用等宽字体并有背景区分
- [ ] 辅助文本（caption/hint）字号不会过小（≥12sp）

---

## 四、图标与图像 (Icon & Image)

- [ ] 图标风格统一（全部 outlined 或全部 filled）
- [ ] 图标尺寸规范（16/24/32/48px）
- [ ] 图标与相邻元素的间距合理（≥8px）
- [ ] 所有图片资源有替代文本（alt）
- [ ] 图片加载失败时显示占位符（非空白）
- [ ] 图片宽高比保持一致（避免布局跳动）
- [ ] 列表缩略图尺寸统一
- [ ] 头像/用户图标有默认占位图
- [ ] 无拉伸或变形的图片
- [ ] 高DPI屏幕下图标清晰（使用 SVG 或 @2x/@3x）

---

## 五、交互与反馈 (Interaction & Feedback)

- [ ] 可点击元素最小 touch target ≥ 44x44px
- [ ] 按钮/链接点击有视觉反馈（ripple/color change）
- [ ] 按钮点击后防止重复提交（debounce/disable）
- [ ] 表单输入有即时反馈（inline validation）
- [ ] 表单错误提示位置与输入框关联（不遮挡输入）
- [ ] 错误提示文案用户友好（非技术术语）
- [ ] 必填项有明确标识（* 或 "Required" 文字）
- [ ] 长按操作有视觉提示（context menu preview）
- [ ] 下拉刷新和上拉加载有明确状态指示
- [ ] 页面切换/加载过程中有反馈（spinner/skeleton/progress）

---

## 六、导航与信息架构 (Navigation & IA)

- [ ] 全局导航入口一致且可预测
- [ ] 底部/侧边导航栏选项不超过 5 个
- [ ] 当前页面在导航中有高亮标识
- [ ] 深层页面有面包屑导航
- [ ] 后退按钮行为符合预期（返回上一页而非首页）
- [ ] 页面刷新后保持当前位置
- [ ] 标签页切换保留各标签状态
- [ ] 搜索功能支持实时搜索（debounce ≥ 300ms）
- [ ] 搜索无结果时有友好提示和建议
- [ ] 添加了 "Scroll to Top" 浮动按钮

---

## 七、动效与过渡 (Animation & Transition)

- [ ] 页面切换动画流畅（60fps）
- [ ] 动画时长合理（页面切换 250-350ms，微交互 100-200ms）
- [ ] 尊重系统 "Reduce Motion" 偏好设置
- [ ] 模态框弹出和关闭动画对称
- [ ] 列表项入场动画简洁（无过度装饰）
- [ ] 拖拽/滑动操作有物理感和弹性
- [ ] 动画未造成性能问题（主线程未阻塞）
- [ ] loading 状态使用骨架屏而非单纯 spinner
- [ ] 状态切换有渐变过渡（非突变）
- [ ] 动画未造成内容跳动或FOUC

---

## 八、组件规范 (Component Standards)

- [ ] 按钮样式统一（primary/secondary/ghost/danger）
- [ ] 输入框样式统一（normal/focus/error/disabled）
- [ ] 卡片有统一的 padding 和圆角规范
- [ ] Toast/Snackbar 有最大显示数量限制（≤2）
- [ ] Modal 有统一的关闭方式（点击外部/点击X/滑动）
- [ ] 下拉菜单/选择器有键盘导航支持
- [ ] 开关 Toggle 有明确的 ON/OFF 状态
- [ ] 进度条有明确的百分比或状态标签
- [ ] 表格/列表支持横向滚动（多列时）
- [ ] 分页器显示当前页和总页数

---

## 九、无障碍与包容性 (Accessibility & Inclusivity)

- [ ] 所有图片和图标有替代文本
- [ ] 表单输入有关联的 label 标签
- [ ] 支持键盘导航（Tab / Shift+Tab / Enter / Esc）
- [ ] 焦点状态有明确的可视化指示
- [ ] ARIA 标签正确使用（role, aria-label, aria-describedby）
- [ ] 颜色不是传递信息的唯一方式
- [ ] 支持动态字体大小调整（不破坏布局）
- [ ] 支持系统语言设置（左-to右 / 右-to左）
- [ ] 视频/音频内容有字幕或文字记录
- [ ] 操作倒计时有延长的替代方案

---

## 十、内容与文案 (Content & Copy)

- [ ] 按钮文案使用具体动词（"发布" 而非 "提交"）
- [ ] 错误提示提供解决方案（而非仅描述错误）
- [ ] 空状态有引导性文案和行动按钮
- [ ] 确认对话框使用明确的行动描述（"确认删除" 而非 "确定"）
- [ ] 提示文案与实际操作一致（无误导）
- [ ] 数字使用本地化格式（千分位/小数点）
- [ ] 日期时间使用用户时区
- [ ] 专业术语有解释或使用通用替代词
- [ ] 版权和法务信息位置正确且完整
- [ ] 语言切换后 UI 布局正常（如中英文混合）

---

## 十一、性能与稳定性 (Performance & Stability)

- [ ] 首屏加载时间 ≤ 3秒（3G网络）
- [ ] 图片懒加载已实现
- [ ] 骨架屏/渐进式加载已实现
- [ ] 网络错误有重试机制
- [ ] 离线状态有友好提示
- [ ] 大列表使用虚拟滚动（≥100项时）
- [ ] 无内存泄漏风险（事件监听器已清理）
- [ ] App 在后台时状态正确保存
- [ ] 长时间使用无性能衰减
- [ ] 无崩溃/ANR 问题报告

---

## 十二、合规与隐私 (Compliance & Privacy)

- [ ] 用户协议和隐私政策链接可访问
- [ ] 权限请求有明确的用途说明
- [ ] 敏感操作需要二次确认
- [ ] 表单不收集非必要个人信息
- [ ] Cookie/本地存储有用户同意提示
- [ ] 密码输入框可切换显示/隐藏
- [ ] 安全输入框禁用自动填充建议
- [ ] 退出登录有确认提示
- [ ] 账户删除有明确的流程和提示
- [ ] 第三方 SDK 集成符合隐私规范
