## 1. 文档目的与范围

### 1.1 目的
本报告使用 **STRIDE 威胁建模方法** 对夏日萤火 iOS 应用进行系统性安全分析，识别所有模块的攻击面，评估潜在威胁，并提供对应的缓解措施。

### 1.2 系统概述
- **应用名称**：夏日萤火 / A-Single-Spark
- **类型**：iOS 原生离线 Mesh 自组织网络通讯应用
- **核心技术**：蓝牙/WiFi P2P、Secure Enclave、E2E 加密、离线地图
- **目标用户**：户外运动爱好者、紧急情况通讯需求用户

### 1.3 建模方法
- **框架**：STRIDE（Microsoft 威胁建模方法）
- **攻击面分析**：覆盖所有 7 个业务模块 + 4 层架构
- **信任边界**：设备本地 ↔ Mesh 网络 ↔ 其他节点

## 3. STRIDE 威胁分析

### 3.1 STRIDE 分类说明

| 字母 | 威胁类型 | 描述 | 对应安全属性 |
|------|---------|------|-------------|
| **S** | Spoofing (欺骗) | 冒充他人身份 | 身份认证 |
| **T** | Tampering (篡改) | 修改数据或代码 | 数据完整性 |
| **R** | Repudiation (抵赖) | 否认操作行为 | 不可否认性 |
| **I** | Information Disclosure (信息泄露) | 暴露敏感信息 | 保密性 |
| **D** | Denial of Service (拒绝服务) | 系统不可用 | 可用性 |
| **E** | Elevation of Privilege (权限提升) | 未授权访问 | 授权 |

---

### 3.2 Identity 模块威胁分析

#### 模块职责
- UID 生成与管理
- 公钥私钥对生成（Secure Enclave）
- Keychain 管理

#### 数据流

```
[用户输入] → [UIDGenerator] → [DeviceIdentity] → [KeychainService] → [Secure Enclave]
                                          ↓
                                    [Mesh广播]
```

#### STRIDE 威胁

| ID | 类型 | 威胁描述 | 攻击路径 | 缓解措施 | 严重性 |
|----|------|---------|---------|---------|--------|
| ID-S1 | S | 攻击者伪造他人 UID 加入 Mesh | 在节点发现阶段广播已知 UID | 证书链机制 + 验签 | 严重 |
| ID-S2 | S | 攻击者通过暴力枚举 UID 试探 | 批量生成 UID 尝试连接 | 连接频率限制 + 异常检测 | 高 |
| ID-T1 | T | 攻击者篡改 Keychain 中的私钥 | 设备越狱后修改 Keychain | 生物认证保护 (`biometryCurrentSet`) | 严重 |
| ID-T2 | T | 攻击者替换公钥关联的 UID | 中间人修改广播内容 | 公钥-UID 绑定签名验证 | 严重 |
| ID-I1 | I | 日志输出暴露 UID 和公钥 | 调试日志被提取 | 生产环境日志脱敏 | 低 |
| ID-R1 | R | 用户否认发布公钥行为 | 争议场景下无法举证 | 签名不可否认性（ECDSA） | 中 |

### 3.3 Mesh 模块威胁分析

#### 模块职责
- 蓝牙/WiFi P2P 连接管理
- 节点发现与路由表维护
- 多跳中继转发

#### 数据流

```
[广播发现] → [MCNearbyServiceBrowser] → [MCPeerID验证] → [路由表更新]
                                              ↓
                                    [MCSession数据传输]
                                              ↓
                                    [中继节点转发]
```

#### STRIDE 威胁

| ID | 类型 | 威胁描述 | 攻击路径 | 缓解措施 | 严重性 |
|----|------|---------|---------|---------|--------|
| MH-S1 | S | Sybil 攻击：攻击者伪造大量虚假节点 | 广播多个伪造 MCPeerID | 证书链 + 资源证明（电池/计算） | 高 |
| MH-S2 | S | 路由表污染：虚假最短路径通告 | 恶意节点广播优化 hop_count | 源节点 hop_count 锁定 + 中继签名 | 高 |
| MH-T1 | T | 消息内容篡改 | 中继节点修改转发数据 | 端到端签名验证 | 高 |
| MH-T2 | T | 路由表条目修改 | 本地数据库被篡改 | SQLite 加密 + 完整性校验 | 中 |
| MH-R1 | R | 节点否认转发行为 | 积分争议时无法举证 | 中继操作带签名日志 | 中 |
| MH-I1 | I | 节点列表信息泄露 | 路由表被读取 | 敏感字段加密存储 | 中 |
| MH-D1 | D | 蓝牙广播干扰（Denial of Service） | 持续广播干扰附近设备 | 跳频策略 + 黑名单机制 | 中 |
| MH-D2 | D | 路由表泛洪攻击 | 大量路由更新耗尽内存 | 路由表大小限制 + 老化机制 | 中 |
| MH-E1 | E | 恶意节点获取管理权限 | 伪造管理员身份发布指令 | 所有管理操作需验签 | 高 |

### 3.4 Crypto 模块威胁分析

#### 模块职责
- E2E 加密/解密
- 消息签名与验签
- 密钥派生与管理

#### 数据流

```
[业务数据] → [E2EEncryptionService.encrypt] → [加密数据] → [Mesh传输]
                                                      ↓
[加密数据] → [E2EEncryptionService.decrypt] → [解密数据]
                                                      ↓
[消息] → [SignerService.sign] → [签名] → [验签]
```

#### STRIDE 威胁

| ID | 类型 | 威胁描述 | 攻击路径 | 缓解措施 | 严重性 |
|----|------|---------|---------|---------|--------|
| CR-S1 | S | ECDSA 签名伪造 | 攻击者生成他人公钥对应的私钥 | 依赖 Secure Enclave 不可导出私钥 | 高（理论上低，实际依赖实现） |
| CR-T1 | T | 密文篡改后验签通过 | 修改密文某字节侥幸通过 | AEAD 模式（GCM）完整性保护 | 高 |
| CR-T2 | T | 密钥派生过程被篡改 | 修改盐值或迭代次数 | 固定 KDF 参数 + 版本校验 | 中 |
| CR-I1 | I | 密钥在内存中被读取 | 冷启动攻击提取内存 | 敏感操作后立即清除内存 | 中 |
| CR-I2 | I | 公钥交换过程泄露会话密钥 | DH 密钥交换被截获 | ECDH + Forward Secrecy | 高 |
| CR-D1 | D | 加密服务不可用（算法错误） | 非法输入导致加密失败 | 输入校验 + 异常捕获 | 低 |
| CR-E1 | E | 私钥被用于非授权操作 | 恶意 App 调用加密服务 | 每次操作需用户生物认证 | 高 |

| VO-S2 | S | 语音来源伪装（中间人） | 截获并重放语音流 | 序列号 + 时间戳防重放 | 高 |
| VO-T1 | T | 语音数据篡改 | 中继节点修改音频包 | E2E 签名验证 | 高 |

| VO-D1 | D | 语音通话被阻断 | 干扰蓝牙/WiFi 信号 | 切换到备用通道 | 中 |
| VO-D2 | D | 编解码器崩溃（Denial of Service） | 畸形音频数据 | 输入长度校验 + 异常隔离 | 中 |

## 4. 攻击面分析

### 4.1 攻击面矩阵

| 模块 | 暴露接口 | 入口点 | 信任级别 | 风险等级 |
|------|---------|--------|---------|---------|
| Identity | 公钥广播、Keychain 访问 | Mesh 发现、App 启动 | 中（网络）/ 高（本地） | 高 |
| Mesh | MCSession、MCPeerID、路由广播 | 蓝牙/WiFi 广播接收 | 低（不可信网络） | 严重 |
| Crypto | encrypt/decrypt/sign/verify API | 所有业务模块调用 | 高（内部） | 中 |
| Voice | 麦克风输入、音频流输出 | 通话建立、数据传输 | 低（网络）/ 中（本地） | 高 |
| Map | 地图瓦片加载、位置输入 | 文件系统、Mesh 传输 | 中 | 中 |
| Points | 积分计算 API、余额查询 | 内部调用 | 高（内部） | 中 |
| Storage | SQLite 查询、Keychain 读写 | 所有持久化操作 | 高（本地） | 中 |

### 4.2 高风险攻击向量

#### 攻击向量 #1：Mesh 节点发现注入
```
攻击路径：蓝牙/WiFi 广播 → MCPeerID 伪造 → 证书链伪造 → 加入 Mesh 网络
利用漏洞：VULN-001（节点发现无零知识验签）
影响范围：所有后续通讯
严重性：严重
```

#### 攻击向量 #2：语音流重放
```
攻击路径：录制语音 → 重放加密包 → 时间戳绕过 → 语音混淆
利用漏洞：VULN-003（重放攻击防护缺失）
影响范围：语音通话、积分计算
严重性：高
```

#### 攻击向量 #3：积分计算欺诈
```
攻击路径：伪造 hop_count → 签名伪造 → 积分记录 → 积分通货膨胀
利用漏洞：VULN-008（积分计算无防篡改）
影响范围：积分体系公平性
严重性：高
```

#### 攻击向量 #4：数据库提取
```
攻击路径：设备越狱 → 解密备份 → 提取 SQLite → 读取积分/路由信息
利用漏洞：VULN-004（密钥管理薄弱）
影响范围：所有本地数据
严重性：高
```

## 5. 缓解措施汇总

### 5.1 按 STRIDE 分类的缓解措施

| 威胁类型 | 缓解措施 | 对应模块 |
|---------|---------|---------|
| **Spoofing** | 证书链机制、身份验签、生物认证保护 | Identity、Mesh |
| **Tampering** | E2E 签名、AEAD 加密、完整性校验 | 所有模块 |
| **Repudiation** | ECDSA 签名、不可篡改日志 | 所有模块 |
| **Information Disclosure** | AES-256 加密、日志脱敏、Keychain 保护 | Storage、Shared |
| **Denial of Service** | 输入校验、异常隔离、流量控制 | 所有模块 |
| **Elevation of Privilege** | 最小权限、Keychain 访问控制、生物认证 | Identity |

### 5.2 安全设计原则

1. **纵深防御**：每个层级都有独立的安全机制
2. **零信任**：Mesh 网络中所有节点都不可信，必须验签
3. **最小权限**：Keychain 和加密操作按需授权
4. **安全默认值**：默认加密、默认验签、默认脱敏
5. **前向保密**：每次会话使用临时密钥

## 6. 风险评估矩阵

### 6.1 风险计算

```
风险等级 = 可能性 × 影响程度

可能性等级：
- 高 (H)：攻击路径明确，实现容易
- 中 (M)：需要特定条件或中间人位置
- 低 (L)：需要越狱设备或物理访问

影响程度等级：
- 严重 (S)：私钥泄露、身份伪造、大规模数据泄露
- 高 (H)：通讯被窃听、积分欺诈
- 中 (M)：信息泄露、服务中断
- 低 (L)：轻微隐私风险
```

### 6.2 风险矩阵

| 风险 ID | 威胁描述 | 可能性 | 影响 | 风险等级 | 状态 |
|--------|---------|--------|------|---------|------|

| R-02 | 语音内容窃听 | M | S | 🟠 高 | 需修复 |
| R-03 | 积分欺诈 | H | H | 🟠 高 | 需修复 |
| R-04 | 数据库泄露 | L | H | 🟡 中 | 需修复 |
| R-05 | 地图数据篡改 | M | H | 🟠 高 | 需修复 |
| R-06 | 路由表污染 | H | M | 🟡 中 | 持续监控 |
| R-07 | 设备越狱 Keychain 提取 | L | S | 🟡 中 | 需修复 |

## 7. 验证与测试建议

### 7.1 安全测试用例

| 测试 ID | 测试类型 | 测试场景 | 预期结果 |
|--------|---------|---------|---------|
| SEC-TT-01 | 身份伪造测试 | 伪造他人 UID 尝试加入 Mesh | 连接被拒绝 |
